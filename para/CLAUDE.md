# PARA

For the full methodology, see `resources/para-methodology.md`. This file covers how PARA maps to this repo.

## Structure

| Folder       | Purpose                                              |
| ------------ | ---------------------------------------------------- |
| `projects/`  | Active work with a specific outcome and done state   |
| `areas/`     | Ongoing responsibilities, maintained to a standard   |
| `resources/` | Reference material, shareable knowledge              |
| `archive/`   | Completed or retired items from the other categories |

## Rules

- Organise by actionability, not by subject
- Every piece of information lives in exactly one place — never duplicate
- Don't create empty folders — structure emerges from use
- When a project is done, move it to archive
- When something stops being your responsibility, archive it
